## 1. Docker Crash Course

The most common issue in software engineering is when your app fails to compile on a different computer even 
when it runs perfectly on yours. Docker solve this issue by allowing you to package an application and all its 
dependencies into a **container**. 

This container can run on any computer with Docker installed, regardless of the host's operating system or other 
system configurations. This allows everyone on your team to run the same setup, preventing conflicts between projects.

But more importantly, it allows you to move between laptops, labs, or robots without having to reinstall any software.

> 📘 Learn more: [Docker Overview](https://docs.docker.com/get-started/overview/)

### 1.2. Getting Hands-On with Docker

- [Install Docker for your OS](https://docs.docker.com/get-docker/)  
- Try running the following commands on your terminal:

```bash
# Download an existing image from Docker Hub
docker pull ubuntu

# Build a custom image from a Dockerfile
docker build -t my_image .

# Run an interactive container
docker run -it my_image

# List running containers
docker ps

# Start a container
docker start -ai <container-id>

# Stop a container
docker stop <container-id>

# Remove a container
docker rm <container-id>
```

## 2. ROS2 Crash Course

The Robot Operating System, or ROS2 is an open-source framework for building sophisticated robot software.

If you have previously worked with Arduino microcontrollers, Raspberry Pis, or Jetson Nano boards, you would be familiar 
with wiring sensors, motors, cameras to communicate with each other. We call this data exchange telemetry, and it takes 
quite a bit of effort to achieve this without bugs creeping in.

ROS2 solves this problem by introducing modular 'nodes' that share data effortlessly via the publish-subscribe 
messaging. This allows you to build safe, predictable control systems for real-world autonomous robots.

📘 Learn more: [ROS2 Overview](https://docs.ros.org/en/rolling/index.html)
